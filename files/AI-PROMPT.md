# Worklog AI Prompt 设计

本项目的核心能力包括：

1. **提交前**：根据 **diff（无 commit message）** 由 AI 生成 **commit message**，用户确认后再提交。
2. **提交后**：将 **Git Commit + Diff** 自动生成 **工作日报 / 周报 / 月报**。

因此 Prompt 设计非常关键（commit message 与日报可用不同模板：前者偏技术摘要与约定式提交，后者偏工作汇报与非技术可读）。

---

# 输入数据结构

AI 输入内容包括：

```
commit message
commit diff
文件变更
```

示例：

```
commit: fix login bug

diff:
src/login.ts
- fix token validation
- add error handling
```

---

# Prompt 设计目标

生成的内容必须：

1. 简洁
2. 适合日报
3. 非技术人员也能看懂
4. 不暴露具体代码细节
5. 支持多语言输出，但**始终保证有一份完整的中文版本（标题 + 内容）**

---

# Prompt 模板

默认中文模板（不带语言参数时）：

```
你是一名软件工程师助理。

任务：
根据 Git commit 和代码变更生成工作日报。

要求：
1. 使用简洁中文，**面向非技术同事和管理者也能看懂**
2. 每条以“完成了 / 实现了 / 新增了 / 更新了 / 优化了 / 提升了 / 解决了”等**成果导向动词**开头，突出“做了什么 + 带来了什么效果/价值”
3. 不要输出代码，不堆砌技术细节或内部术语（如“占位符、draft、临时标记”等）
4. 如需提到技术名词，请简单说明用途，例如“支持多语言报告输出，方便对外同步和跨团队沟通”
5. 每条一句话，不超过5条

输入：

commit message:
{commit_message}

code diff:
{diff}

输出格式：

今日工作：

1.
2.
3.
```

多语言扩展模板（显式指定 language 时）：

```
你是一名软件工程师助理。

任务：
根据 Git commit 和代码变更生成工作日报。

要求：
1. 始终生成一份简洁的**中文日报**（标题 + 内容），适合发给中国主管和非技术同事阅读
2. 每条以“完成了 / 实现了 / 新增了 / 更新了 / 优化了 / 提升了 / 解决了”等成果导向动词开头，突出“做了什么 + 带来了什么效果/价值”
3. 避免使用内部术语，如“占位符、draft、临时标记”等；如果是规划未来工作，请表述为“规划了后续 XX 功能，并在计划中增加了相关事项”
4. 如果 language 不为 "zh" ，在中文日报后面再生成一份目标语言 {language} 的等价日报，含义尽量与中文版本对齐
5. 不要输出代码，每条一句话，不超过5条

输入：

language: {language}  // 例如 "en"、"ja"

commit message:
{commit_message}

code diff:
{diff}

输出格式（示例）：

中文日报：

今日工作：
1.
2.

{language} 日报：

Today:
1.
2.
```

---

# 示例

输入：

```
commit message:
fix login bug

diff:
login.ts
add token validation
```

输出：

```
今日工作：

1. 修复登录模块中的认证问题
2. 优化用户登录流程稳定性
```
